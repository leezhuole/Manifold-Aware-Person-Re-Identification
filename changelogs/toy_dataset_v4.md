# Toy-Dataset Randers Validation Framework — Plan

## Context

The user's intuition (Intuition 1.pdf) proposes a minimal Randers-drift model anchored on the synthetic ToyCorruption dataset (`examples/data/ToyCorruption`, generated by `scripts/generate_toy_dataset.py`): two clean source crops per PID (distinct cameras) × five monotonic ImageNet-C-style severities. Past diagnostics in `bau/` (absorption ratios, cross-source matrices) are described by the user as ad-hoc and message-confusing. This plan replaces them with a **clean, two-claim validation** in `old_bau/` so the asymmetric story is decoupled from the `bau/` Finsler edits already in flight.

The two claims to be proven:

- **(A) Task asymmetry under the toy protocol** — query (clean, σ=0) vs. gallery (mixed σ=1..4) is not symmetric: swapping query/gallery roles changes mAP/Rank-k. This is *not* a property of person ReID in general (see critique below); it is a property of *this* protocol, which is the point.
- **(B) 1-D corruption captured by ω + extra feature dim** — appending one feature dim `z_w` to a 2048-d ResNet-50 BAU baseline and tying a single global ω ∈ ℝ^{D+1} to it via two auxiliary losses suffices to (i) align ω with the appended axis, (ii) make `z_w` monotone in severity, (iii) recover an asymmetric retrieval signal of the right sign at evaluation.

## Critical evaluation of the user's intuition (peer-level, not deferential)

These are the load-bearing assumptions and how each survives literature scrutiny.

1. **"ReID is inherently asymmetric"** — *Overclaim.* Standard DG-ReID (BAU, TransReID, Market/MSMT/CUHK protocols) uses symmetric Euclidean/cosine distance and is empirically symmetric in expectation. Asymmetry arises from **query/gallery covariate shift** (cross-resolution: Li CVPR'19; cross-view: AG-ReIDv2 Nguyen TIFS'24; cross-domain: Budnik & Avrithis ICCV'21), not from the task. Scope the claim: *under the toy protocol where query=clean and gallery is severity-shifted, retrieval is structurally asymmetric.*

2. **"ω ∈ ℝ^{D+1} sampled from 𝒩(0,1)"** — *Inconsistent with quasi-metric correctness.* For `d_FC(x,y) = ‖y−x‖₂ + ω^⊤(y−x)` to be a valid (quasi-)metric (positivity + triangle inequality), Bao–Chern–Shen (2000, Ch. 11) require `‖ω‖₂ < 1` strictly. An i.i.d. 𝒩(0,1) draw in ℝ^{2049} has `‖ω‖ ≈ √2049 ≈ 45`, which violates the condition by ~45×. **ω must be norm-gated** (e.g., `‖ω‖ ≤ 0.95`) at initialization and throughout training. The repo's existing `resnet50_finsler` already gates at 0.95; reuse that constant.

3. **"L_1 zero-pads identity displacement"** — *Mechanically clever, but tautological if interpreted naively.* Zero-padding `(z_i^σ − z_j^σ, 0)` and forcing `⟨…, ω⟩ = 0` constrains only `ω_{1:D}`, leaving `ω_{D+1}` unconstrained. Combined with L_2 driving `z_w` to grow with σ, training is incentivized to push ω's mass onto the appended axis. Verify this empirically: track `‖ω_{1:D}‖ / ‖ω‖` over training; it should decrease monotonically. If it does not, the orthogonality assumption is violated by identity geometry (i.e., identity displacements are not isotropic in ℝ^D).

4. **"Single global ω is enough because severity is monotonic"** — *Conditional.* This is true only if the six composed corruptions (defocus + motion + bilinear down/up + gamma + shot/Gaussian + JPEG) push features in approximately collinear directions in ℝ^D. ImageNet-C literature (Hendrycks & Dietterich, ICLR'19) treats them as orthogonal stress dimensions, not collinear. **Pre-flight test required (D1 below)** — if the per-corruption displacements are non-collinear, `η_B` (the 1-D absorption ratio) is upper-bounded by the cosine² of the angle between the true degradation manifold and ω, and the simple intuition is capacity-limited.

5. **Real-world analogue claim** — see next section. The toy ladder has **no clean real-world analogue** that satisfies strict 1-D monotonicity; the closest is BRIAR-style long-range biometrics (range as a scalar), and the closest *methodological* precedent is the Corrupted-ReID benchmark (Chen et al., arXiv:2206.02416) which is itself synthetic.

## Answer: "In which real-world scenario would a ReID crop progressively deteriorate as in the toy dataset?"

**Best-defensible analogue: long-range biometrics under atmospheric turbulence.** The IARPA BRIAR program collects subjects at standoff distances (≈100–500+ m); turbulence intensity scales with range and the refractive-index structure parameter C_n², producing approximately monotonic degradation in blur/scintillation/geometric warp. References: Cornett et al. WACV-W'23 ("Expanding Accurate Person Recognition to New Altitudes and Ranges: The BRIAR Dataset"); Vorontsov & Carhart imaging-through-turbulence work cited within. Caveat: turbulence is dominantly *anisoplanatic blur + warp*, not the composed pipeline used here, so the analogy is on the **scalar-severity axis**, not on the operator mix.

**Closest methodological precedent: Corrupted-ReID / Person-ReID-C** (Chen, Liao et al., arXiv:2206.02416, "Benchmarks for Corruption Invariant Person Re-identification"). They apply ImageNet-C corruptions at discrete severities to Market/MSMT/CUHK; this is what the toy dataset *operationalises*. It is synthetic — it validates the *protocol*, not ecological deployment.

**Failed candidates for the strict 1-D test:**
- AG-ReIDv2 (Nguyen TIFS'24): altitude is monotonic but degradation is multi-modal (resolution + viewpoint + background), not single-axis.
- Compressed-video surveillance: bitrate is scalar but adaptive encoding (QP per frame) breaks monotonicity in time.
- Weather/haze ReID: optical depth is approximately scalar, but rain/snow/illumination break ordering; no benchmark provides a controlled scalar haze ladder for ReID.

**Recommended framing in the paper:** cite BRIAR/Cornett'23 for ecological grounding of "scalar quality axis" + Corrupted-ReID/Chen'22 for methodological precedent + Hendrycks ICLR'19 for the operator taxonomy. Explicitly state the toy ladder is a *stylised* axis, not a calibrated ISP simulation (the synthesis doc already notes this).

## User-confirmed design choices (2026-04-26)

- **ω is globally fixed at initialization** (not learnable). One i.i.d. draw `ω ~ 𝒩(0, I_{D+1})` normalised to `‖ω‖ = 0.95`, registered as a frozen buffer (not `nn.Parameter`).
- **L_1 zero-pads** the identity displacement: `L_1 = (⟨pad(z_i^σ − z_j^σ), ω⟩)²`.
- **Train and evaluate on the toy dataset.** Market-1501 has no severity axis, so L_1 / L_2 would be statistically undefined. This requires expanding [scripts/generate_toy_dataset.py](scripts/generate_toy_dataset.py) to emit a large *training* split alongside the existing 50-PID *evaluation* split.
- **Asymmetry test = clean→corrupted vs corrupted→clean** mAP/Rank-1 gap.

### One critical geometric concern raised by the "globally fixed random ω" choice

With `D = 2048` (BAU baseline), an isotropic 𝒩(0, I_{D+1}) draw normalized to 0.95 distributes mass uniformly across components: `E[|ω_k|²] = 0.95²/(D+1) ≈ 4.4·10⁻⁴`, i.e. the appended axis carries only `|ω_{D+1}| / ‖ω‖ ≈ 1/√{D+1} ≈ 2.2%` of ω's energy. Consequence in `d_FC = ‖y−x‖₂ + ω^⊤(y−x)`:

- The `ω_{D+1}·(z_w_y − z_w_x)` term — the channel the user *wants* to carry corruption — is ~45× weaker than the `ω_{1:D}^⊤(y_{1:D} − x_{1:D})` term.
- For the asymmetric signal to be dominated by `z_w`, the model must inflate `‖z_w‖` to ~45× the typical identity displacement scale, which the optimization is unlikely to do under a hinge-margin L_2.

**This does not break the framework but changes how it succeeds.** Under fixed-random ω in 2049-D, the asymmetry will be picked up *primarily by the original 2048-D subspace aligning corruption shifts with `ω_{1:D}`*, with `z_w` providing a small auxiliary contribution. That is an empirically valid outcome — but it means the headline claim "the *appended* dimension captures corruption" is at risk of being overturned. To pressure-test this honestly, the framework runs in two ω-init regimes side-by-side, both globally fixed:

- **Regime A (isotropic, as user specified):** `ω ~ 𝒩(0, I_{D+1}) / ‖·‖ × 0.95`. Tests whether L_1 + L_2 + the appended axis are *redundant* with R^D directional learning.
- **Regime B (mass-concentrated):** `ω = 0.5 · e_{D+1} + (0.5/√D) · 𝒩(0, I_D)` with all of it normalized to 0.95. Roughly 50/50 split between the appended axis and a random R^D direction. Tests whether the appended-axis claim holds when the geometry *gives it a chance*.

This costs only one extra training run and resolves the most likely review objection ("why does adding +1 dim matter if ω hardly weights it?").

## Phase 0 — Diagnostic battery on Euclidean baseline (run BEFORE adding any drift machinery)

Run on the existing `old_bau` ResNet-50 trained on the **expanded toy training set** (see Phase 0.5 below) and evaluated on the existing 50-PID toy split (no retraining needed for these diagnostics themselves; they consume only frozen features). All four diagnostics live in a new file `old_bau/BAU/scripts/toy_diagnostics_baseline.py` and consume `examples/data/ToyCorruption/metadata.json`.

- **D1 — Per-corruption displacement collinearity (tests assumption #4).** For each individual corruption operator (apply blur-only, motion-only, JPEG-only, etc., at the highest severity) compute mean displacement `Δ_op = E_PID[ z(x_op) − z(x_clean) ]`. Compute the cosine matrix between the six `Δ_op`s. **Predicted outcome**: if cosines ≥ 0.5, the 1-D ω model is plausible; if cosines < 0.2, the user must commit to ω ∈ ℝ^k for k > 1 (multi-direction drift) or restrict the toy to a single-operator ladder. *This is the load-bearing pre-flight check.* Reuses `bau/loss/triplet.py:euclidean_dist` for distance sanity.
- **D2 — Severity monotonicity in the displacement direction.** Define `ω̂ = PC1 of stacked {z(x^σ) − z(x^0) : PID, σ}`. Plot `μ_σ = mean ⟨z(x^σ) − z(x^0), ω̂⟩` for σ ∈ {0..4}. **Predicted**: monotone increase. If non-monotone, the synthesis pipeline does not yield a monotone severity-as-scalar.
- **D3 — Identity-displacement orthogonality (tests L_1's premise).** For pairs (i, j) at fixed σ, histogram `⟨z_i^σ − z_j^σ, ω̂⟩ / (‖…‖·‖ω̂‖)`. **Predicted**: zero-centered, narrow. If the histogram is biased, `ω̂` is leaking into identity directions and L_1 will pull against identity discrimination.
- **D4 — Asymmetric retrieval baseline.** Compute mAP/Rank-1 in two retrieval directions on the Euclidean baseline:
  - *clean→corrupted*: query = `query_s1` (clean, source 1), gallery = `gallery` (rest)
  - *corrupted→clean*: query = severity-4 source-1 crops, gallery = `query_s1 ∪ query_s2` (clean anchors)
  Use the existing evaluator (`old_bau/BAU/bau/evaluators.py:pairwise_distance`). **Predicted**: the Euclidean baseline is symmetric in the metric (`d(x,y) = d(y,x)`) but the *task* asymmetry shows up as different mAP/Rank-k due to gallery-pool covariate shift. *This is the empirical foundation for claim (A) — it must hold without any drift machinery.*

If D1 fails (collinearity too low), STOP — the proposed framework is misspecified for this dataset. If D2/D3 are within tolerance, proceed to implementation.

## Phase 0.5 — Expand the toy dataset for training

The existing `examples/data/ToyCorruption` (50 PIDs × 2 sources × 5 severities = 500 images) is fine as an evaluation probe but too small to fit a backbone. Amend [scripts/generate_toy_dataset.py](scripts/generate_toy_dataset.py) to emit a **training split** alongside the existing eval split:

- New CLI flags: `--train-source-dir` (defaults to Market-1501 *bounding_box_train*), `--num-train-identities` (default `750`), `--num-train-sources-per-pid` (default `4`, since training PIDs in Market typically have ≥4 cameras).
- New output subtree: `examples/data/ToyCorruption/train/{bounding_box_train,query,gallery}` mirroring Market-1501 layout, with the same severity-encoding-in-seq-id filename convention.
- Same RNG keying (sha256 of `(seed, orig_pid, source_idx, severity)`) so re-runs are deterministic and the train/eval splits stay disjoint.
- Update `metadata.json` to carry both `train` and `eval` blocks; bump `TOY_DATASET_VERSION` to `"4.0"`.

Total training-set size at the proposed defaults: 750 PIDs × 4 sources × 5 severities = **15 000 images**, a workable scale for ResNet-50 fine-tuning. Train PIDs are disjoint from the 50 eval PIDs by construction (different Market splits).

**This is the only edit to the `scripts/` tree;** all model/loss/eval changes stay inside `old_bau/BAU/`.

## Phase 1 — Implementation in `old_bau` (minimal surgical edits)

Goal: keep the Euclidean BAU baseline untouched as a control; add an opt-in `--use-randers` flag that turns on (a) +1 output dim, (b) global ω, (c) two auxiliary losses, (d) Randers eval distance. Each edit is named, scoped, and reversible.

**Files to modify:**

1. [old_bau/BAU/bau/models/model.py](old_bau/BAU/bau/models/model.py)
   - Add `extend_dim: bool = False` and `register_omega: bool = False` kwargs to the `resnet50` constructor.
   - When `extend_dim=True`: insert a `nn.Linear(2048, 2049, bias=False)` projection after the BN-neck (post-line ~48), initialised so the first 2048 outputs are ≈ identity (block-identity-plus-zero) and the new axis starts at 0. Forward returns `(emb_2049, normalized_emb_2049, logits_unchanged)`. The classifier head stays on the original 2048-d BN-neck output to *isolate* the new axis from ID classification.
   - When `register_omega=True`: register `self.omega` as a **frozen buffer** (`self.register_buffer('omega', tensor, persistent=True)`) so it survives checkpoints but never receives gradients. Two init regimes selected by `--omega-init {isotropic,mass-concentrated}` (default `isotropic` per user spec): both produce a fixed unit-mass direction scaled to 0.95. Implementation is ~10 lines; seeded by `--omega-seed` (separate from training seed) so the same direction can be reproduced across runs.

2. [old_bau/BAU/bau/loss/randers.py](old_bau/BAU/bau/loss/randers.py) — *new file*
   - `randers_dist(x, y, omega)` returning `‖y−x‖₂ + ω^⊤(y−x)`, broadcasting over `[N, D+1]` and `[M, D+1]`.
   - `orthogonality_loss(emb_2049, pids, omega)` (= L_1): for each PID, sample identity displacements at the same severity (requires severity in batch — see #4 below), zero-pad the last dim, compute `mean (⟨pad(z_i − z_j), ω⟩)²`. (Square, not norm — differentiable and convex.)
   - `monotonicity_loss(emb_2049, severities)` (= L_2): collect `z_w = emb_2049[:, -1]` and apply a rank-correlation surrogate or pairwise-margin loss `Σ_{σ_a < σ_b} ReLU(z_w^a − z_w^b + margin)`. Pairwise hinge is preferred over Spearman (differentiable, implemented in 5 lines).

3. [old_bau/BAU/bau/datasets/toy_corruption.py](old_bau/BAU/bau/datasets/toy_corruption.py) — *new file*
   - Mirror `bau/datasets/market1501.py` pattern (factory-registered `ToyCorruption` class).
   - Parse `examples/data/ToyCorruption/metadata.json` to surface `severity` per sample as a fourth label alongside `(path, pid, cam_id)`. Two query splits: `query_s1` (source-1 clean) and `query_s2` (source-2 clean) for the asymmetry direction tests.
   - Register key `'toycorruption'` in `bau/datasets/__init__.py`.

4. [old_bau/BAU/bau/utils/data/preprocessor.py](old_bau/BAU/bau/utils/data/preprocessor.py)
   - Extend `Preprocessor.__getitem__` to also return `severity` when present in the dataset tuple. Backward-compat: default to `severity = 0` if absent.

5. [old_bau/BAU/bau/trainers.py](old_bau/BAU/bau/trainers.py)
   - In `_parse_data`, accept optional `severities`.
   - When `--use-randers`: add `loss_ortho = ortho_w * orthogonality_loss(...)` and `loss_mono = mono_w * monotonicity_loss(...)`. Default weights `ortho_w = 0.1`, `mono_w = 0.1` (sweep later). Triplet/CE/align/uniform/domain stay on the **first 2048 dims only** (slice `emb_2049[:, :2048]`) — this guarantees the Euclidean baseline behaviour is preserved when the Randers losses are zero.

6. [old_bau/BAU/bau/evaluators.py](old_bau/BAU/bau/evaluators.py)
   - Add `dist_func` parameter to `pairwise_distance` (default Euclidean).

7. [old_bau/BAU/examples/train_bau.py](old_bau/BAU/examples/train_bau.py)

## Phase 2 — Validation protocol (Phase-1 success criteria)

Train **three** models on the expanded toy training split with identical hyper-parameters except the Randers flag and ω-init regime, then evaluate on the existing 50-PID toy eval split. Models: (i) Euclidean baseline, (ii) Randers Regime A (isotropic ω), (iii) Randers Regime B (mass-concentrated ω). Framework is validated iff **V1, V3, V4** hold; **V2 is reformulated** to reflect that ω is fixed.

- **V1 (claim A, baseline):** `mAP[clean→corrupted]` ≠ `mAP[corrupted→clean]` for the Euclidean baseline (gap > 1 mAP point). Evidence that the *task* is asymmetric.
- **V2 (claim B, geometry alignment — reformulated since ω is fixed):** track which axis carries the corruption signal. Compute `Δ_corruption = E_PID[ z(x^{σ=4}) − z(x^{σ=0}) ]` after training; report `cos(Δ_corruption, ω)` (overall) and `cos(Δ_corruption, e_{D+1})` (appended-axis-only). Regime A passing criterion: `cos(Δ_corruption, ω) ≥ 0.7` (model uses ω somewhere). Regime B passing criterion: additionally `cos(Δ_corruption, e_{D+1}) ≥ 0.5` (model uses the appended axis).
- **V3 (claim B, monotonicity of `z_w`):** Spearman ρ between `z_w` and σ on held-out toy eval samples ≥ 0.8. Evidence the appended dim encodes severity *regardless* of whether ω weights it heavily.
- **V4 (claim A → B coupling):** Randers ranking *closes* the asymmetric mAP gap from V1 by ≥ 50% in at least one regime. Evidence that the right-signed asymmetric distance compensates for the protocol asymmetry.

A negative result on V4 with positives on V1/V3 is *also informative*: it would mean the proposed Randers form has the wrong sign convention or that gallery-pool composition (mixed σ) absorbs the drift. **Comparing Regime A vs Regime B on V2/V4 directly answers the question "does the appended +1 dimension carry weight, or is it geometrically redundant given ω's mass distribution?"**

## Risks and pressure-test items

- **R1 (high):** D1 collinearity may be low — composed corruptions push features in non-collinear directions. Mitigation: pre-flight only the single-operator ladder (e.g., JPEG-only, severities 0..4) before committing to the composed ladder.
- **R2 (medium):** L_1's zero-padding may lock ω onto the appended axis trivially (ω_{1:D} → 0) without learning anything substantive. Mitigation: track `‖ω_{1:D}‖` over training; if it collapses in <10 epochs, the loss is degenerate (uninformative). Consider replacing L_1 with `(z_i^σ − z_j^σ)^⊤ ω` directly (no zero-pad) so all D+1 components participate.
- **R3 (medium):** Randers ranking with a global ω cannot recover *per-PID* asymmetry; it only models a global drift. Acceptable for the toy claim but flags a generalization limit for real datasets.
- **R4 (low):** The toy dataset is small (50 PIDs × 2 sources × 5 severities = 500 images). Statistics may be noisy; report all V1–V4 with bootstrap CIs and seed variability (≥3 seeds).

## Verification (end-to-end)

1. Regenerate toy with training split: `python scripts/generate_toy_dataset.py --num-train-identities 750 --num-train-sources-per-pid 4 --seed 42`.
2. Run baseline training: `python examples/train_bau.py --arch resnet50 -ds toycorruption_train -dt toycorruption_eval --logs-dir logs/baseline_toy`.
3. Run Phase-0 diagnostics: `python scripts/toy_diagnostics_baseline.py --resume logs/baseline_toy/best.pth --out results/toy_diagnostics/baseline/`. Gate on D1 cosines; record D4 asymmetric mAP gap (V1).
4. If D1/D2/D3 pass, run Randers training in *both* regimes: `--use-randers --omega-init isotropic --omega-seed 0` and `--use-randers --omega-init mass-concentrated --omega-seed 0`, both with `--randers-ortho-weight 0.1 --randers-mono-weight 0.1 --randers-eval`.
5. Run Phase-1 diagnostics: `python scripts/toy_diagnostics_randers.py --resume logs/randers_{A,B}/best.pth --out results/toy_diagnostics/randers_{A,B}/`. Check V2/V3/V4 per regime.
6. Report a single comparison table (rows: V1..V4, columns: Euclidean / Regime A / Regime B), seed-mean ± std over ≥3 seeds. This replaces the prior ad-hoc absorption-ratio battery.
